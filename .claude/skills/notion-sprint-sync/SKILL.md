# Notion Sprint Sync

Notion의 Voice Map 프로젝트 데이터를 읽어와 현재 스프린트 상태를 파악한다.

## 언제 사용하는가

- 세션 시작 시: 현재 스프린트의 태스크 상태를 파악
- 세션 복구 시: 중단된 지점부터 재개하기 위한 상태 확인
- 스프린트 플래닝 시: 이전 스프린트 결과 확인 후 다음 계획 수립
- 스프린트 회고 시: 완료/미완료 태스크, 의사결정 로그 취합

## 참조하는 Notion DB

### 04. 버전 관리 (스프린트)
- DB ID: `ea48cbc1387b43f9a2c43a6f048cbf4b`
- 읽어올 정보: 현재 스프린트의 목표, 상태, 기간, 포함 기능

### 06. 서브태스크 (칸반)
- DB ID: `aa3e485d3c7d4a19a6dd2fe7e8d1d492`
- 읽어올 정보: 현재 스프린트에 속한 태스크들의 상태, 담당 Agent, 구현 요약
- 필터: 해당 스프린트에 속한 태스크만 조회

### 05. 의사결정 로그
- DB ID: `c0a23822df13405eb318b4c42079479a`
- 읽어올 정보: 해당 스프린트 기간 내 결정 사항들

## 실행 절차

### 1. 현재 스프린트 확인
Notion MCP로 04. 버전 관리에서 "상태"가 "진행중"인 스프린트를 조회한다.

### 2. 태스크 현황 조회
06. 서브태스크에서 해당 스프린트의 태스크를 전부 조회하고, 상태별로 분류한다:
- 대기: 아직 시작하지 않은 태스크
- 진행중: 현재 작업 중인 태스크
- 리뷰중: 완료 후 리뷰 대기
- 완료: 완료된 태스크
- 블로커: 진행 불가 태스크

### 3. 상태 요약 생성
사용자에게 보여줄 요약을 생성한다:
```
[Sprint N 현황]
목표: {스프린트 목표}
기간: {시작일} ~ {종료일}

진행률: {완료}/{전체} ({퍼센트}%)
├── 대기: N개
├── 진행중: N개
├── 리뷰중: N개
├── 완료: N개
└── 블로커: N개

Agent별 현황:
├── Frontend: N개 완료 / N개 배정
├── Backend: N개 완료 / N개 배정
└── AI Knowledge: N개 완료 / N개 배정
```

### 4. 세션 복구 시 추가 동작
의사결정 로그에서 해당 스프린트의 L2 결정 사항을 조회하여, 팀원 spawn 시 컨텍스트에 포함할 정보를 준비한다.
